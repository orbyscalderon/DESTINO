# i18n — Destino TV

Stack: `i18next` + `react-i18next` + `i18next-browser-languagedetector`.

Bootstrap: [src/lib/i18n.js](src/lib/i18n.js). Importado una vez en `main.jsx`.

Idiomas: `es` (fallback), `en`, `pt`. Detección automática del browser; el user puede cambiar en Settings (persiste en `localStorage` bajo `destino-lang`).

## Namespaces (keys raíz en los JSON)

| Namespace | Cubre | Ejemplo |
|---|---|---|
| `common` | Botones, acciones genéricas, mensajes de loading/error | `common.save`, `common.cancel` |
| `auth` | Login, register, forgot password, OAuth | `auth.login`, `auth.continue_with_apple` |
| `nav` | Bottom nav, side menu, headers de página | `nav.home`, `nav.shows` |
| `home` | Feed swipe, matches | `home.no_more_users` |
| `reels` | Feed reels, comentarios | `reels.write_comment` |
| `studio` | ShowStudio (creator UI) | `studio.go_live`, `studio.private_mode` |
| `live` | LiveShow (viewer UI), countdown privado | `live.viewers_count_one`, `live.private_countdown` |
| `viewer` | Header + acciones en LiveShow (legacy, no añadir más aquí — usar `live`) | `viewer.follow` |
| `dashboard` | Creator dashboard, stats, payouts | `dashboard.balance`, `dashboard.top_tippers` |
| `settings` | Página Settings + bloques (lang, push, 2FA) | `settings.two_factor` |
| `stories` | Feature stories 24h | `stories.your_story` |
| `errors` | Mensajes de error de red/API | `errors.server_error` |

## Patrón de uso

```jsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  return (
    <>
      <h1>{t('home.title')}</h1>
      <button>{t('common.save')}</button>
      <p>{t('live.viewers_count', { count: 42 })}</p>
      {/* pluralización: i18next elige automáticamente entre
          live.viewers_count_one y live.viewers_count_other */}
    </>
  );
}
```

### Interpolación

```jsx
t('live.subscribe_to', { name: 'Ana' });        // "Suscríbete a Ana"
t('live.private_countdown', { seconds: 60 });   // "Show privado en 60s"
```

### Pluralización

Para contables, añade `_one` y `_other` (i18next picks el correcto):

```json
"viewers_count_one":   "{{count}} espectador",
"viewers_count_other": "{{count}} espectadores"
```

```jsx
t('live.viewers_count', { count: viewers.length })
```

## Cómo añadir una nueva traducción

1. Decide el namespace (o añade uno nuevo si no existe).
2. Añade la key en **los tres** archivos: `es.json`, `en.json`, `pt.json`. Los tres tienen que estar sincronizados — un key faltante en uno hace caer al fallback.
3. En el componente:
   ```jsx
   const { t } = useTranslation();
   <button>{t('namespace.key')}</button>
   ```

## Convenciones

- **Strings cortos (botones)** → `common.*` si son verbos genéricos (Guardar, Cancelar, Cerrar)
- **Strings de página** → namespace específico (`home.*`, `studio.*`)
- **Errores de red/API** → `errors.*` para que sean reutilizables
- **No traducir** marcas (Destino TV, LiveKit, Stripe), nombres de usuarios, contenido generado por usuarios
- **Emoji + texto**: el emoji va en el string traducido si tiene sentido cultural (⭐ VIP), no si es decorativo

## Variables que aún no se traducen

Toasts (`react-hot-toast`) y mensajes desde el backend siguen en español. Plan futuro:
- Toasts: extraer a `errors.*` o crear helper `toastError(key)` que use `t()`
- Backend: el backend siempre responde en español por ahora; para localizar, leer `Accept-Language` y devolver código + payload genérico, traducir en frontend
